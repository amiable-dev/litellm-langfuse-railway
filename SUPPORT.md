## Bug Reports and Feature Requests

### Found a Bug?

First, determine if it's a template issue or an upstream issue:

| Symptom | Report To |
|---------|-----------|
| Template won't deploy on Railway | [This repo](https://github.com/amiable-dev/litellm-langfuse-railway/issues/new?template=bug_report.md) |
| Configuration file syntax error | [This repo](https://github.com/amiable-dev/litellm-langfuse-railway/issues/new?template=bug_report.md) |
| LiteLLM runtime bug (crashes, wrong responses) | [LiteLLM Issues](https://github.com/BerriAI/litellm/issues) |
| Langfuse UI bug, tracing issues | [Langfuse Issues](https://github.com/langfuse/langfuse/issues) |
| Railway platform issues | [Railway Support](https://railway.app/help) |

### Want a Feature?

1. Check [existing feature requests](https://github.com/amiable-dev/litellm-langfuse-railway/issues?q=label%3Aenhancement)
2. Open a [feature request](https://github.com/amiable-dev/litellm-langfuse-railway/issues/new?template=feature_request.md)
3. Describe your use case and expected behavior

## Cost Considerations

Remember that deploying these templates creates real resources on Railway:

- Review [Railway pricing](https://railway.app/pricing) before deploying
- Set up spending limits in your Railway dashboard
- Delete unused test deployments promptly
